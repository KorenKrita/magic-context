# v0.25.0

Automatic reclaim of old tool output, clearer reduction guidance that models actually act on, a Pi `/ctx-dream` fix, and a prompt-cache hardening.

## Highlights

- **Old tool output is now reclaimed automatically.** Until now, shrinking a heavy session relied on the agent calling `ctx_reduce` on spent tool results — and models reliably drop *recent* outputs while old ones pile up untouched, so sessions parked with large reclaimable file-reads and command output sitting in context. Magic Context now drops tool output that has survived a full work cycle automatically: once a tool result is old enough that the agent has clearly moved on, it's reclaimed on the next pass that's already updating context — no extra cost, and the recent window is always preserved so nothing the agent is still working with is touched. This is the main lever the execute threshold was missing.

- **`ctx_reduce` guidance models actually follow.** The tool described itself like an irreversible delete ("dropped content is gone forever"), so models — GPT-family especially — hoarded it until the very end of a turn, leaving spent output in context far longer than needed. It's now described accurately: marking content is a *deferred discard* — it stays visible until space is actually needed, the most recent items are protected, and you mark things the moment you're done with them. In testing this made models reclaim spent output noticeably earlier, while keeping the one real caution (don't discard unprocessed evidence). Agents can also now reclaim a large pasted blob inside their own earlier message (logs, dumps, long code) once they've used it — it stays searchable.

## Fixes

- **`/ctx-dream` works on Pi again.** On Pi (and OpenCode Desktop), `/ctx-dream` failed with `Unknown named parameter '0'`. The two SQLite engines Magic Context runs on disagree on one binding style, and a single call used the form only one of them accepts. Fixed at the call site and hardened at the shared database layer so the whole class of mismatch can't recur, with a guard test to keep it from coming back. ([#151](https://github.com/cortexkit/magic-context/issues/151))

- **Clearer handling when local embeddings can't load.** If the on-device embedding runtime (`onnxruntime-node`) is missing or its platform binary failed to install — seen on some Windows setups — Magic Context now reports it once with an actionable fix instead of repeating a cryptic error on every attempt, and `doctor` flags it up front. The fix: reinstall with `doctor --force`, or point `embedding` at an OpenAI-compatible / Ollama endpoint. ([#128](https://github.com/cortexkit/magic-context/issues/128))

- **Prompt-cache hardening.** Dropped content now always renders as a single, fixed placeholder, so a dropped item can never change shape between turns and invalidate the cached prefix after it. The reduction reminders that nudge the agent are also delivered in a way that no longer disturbs the cache.

## Also in this release

- The context-reduction reminders now reach background workers and subagents too, so long-running delegated work manages its own context instead of growing until it overflows.
